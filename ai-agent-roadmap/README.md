## 🎯 核心学习原则

> 这套原则贯穿所有阶段,每个阶段日志开头会简短重申。完整版见 [00-roadmap.md](00-roadmap.md)。

1. **概念优先** — 先讲原理,再给代码,再展开真实场景
2. **AI 边界** — Cursor / Claude 仅作 API 字典,不替我做判断
3. **完成标志硬核检查** — 每章末尾按完成标志逐条问答,不靠"我懂了"过关
4. **挑战数字和前提** — 任何具体数字要问来源,任何判断要问证据
5. **区分判断和事实** — 给意见时明确标注"这是判断,不是事实"

## 🔒 项目隔离原则(不可妥协)

学习项目和 Vertiv PI 工作完全隔离:

- **数据源**:100% 公开(NVD、CVE-search、ISO 27001 公开文档)
- **代码**:100% 个人(不复用任何 PI 代码、配置、内部文档)
- **基础设施**:100% 个人(个人服务器、个人 API key、个人电脑)
- **用途**:100% 学习(不用于商业目的,即使脱敏后也只用于个人简历)
